# API Docs Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add shareable `/docs` and `/docs/:apiId` pages that render MDA OpenAPI docs with a custom GovHub renderer and enforce per-API documentation visibility.

**Architecture:** Backend docs access rules live in focused helpers and a docs router, with schema migration support for `apis.docs_visibility`. Frontend adds a docs index and per-API custom OpenAPI page, then exposes docs and access groups from the sidebar/settings. Existing catalog detail remains intact and links into the docs route.

**Tech Stack:** Express, better-sqlite3, TypeScript, React, Vite, React Router, Tabler icons.
