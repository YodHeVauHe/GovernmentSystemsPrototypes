# Security Policy

## Supported Scope

Uganda GovHub API is a prototype for showcase, procurement, technical, and
security evaluation. It is not approved for production use without separate
written authorization, penetration testing, operational hardening, and a
deployment-specific security review.

## Reporting Security Issues

Report suspected vulnerabilities, leaked credentials, unauthorized access,
license violations, or confidential information exposure privately to the
repository owner.

Do not disclose sensitive findings publicly unless the repository owner has
given written permission or a legal obligation requires disclosure.

## Confidentiality

Security reports, proof-of-concept details, screenshots, logs, credentials, and
repository access details are confidential. Do not share them with anyone who is
not explicitly authorized to review this repository.

## Reviewer Rules

Authorized reviewers may inspect the code for the approved review purpose only.
They may not:

- Attack, scan, or test systems outside the written review scope.
- Use discovered credentials or secrets to access external systems.
- Exfiltrate data or repository contents.
- Publish vulnerability details before coordinated review.
- Reuse source code, system designs, API specifications, or security controls in
  another project without written permission.
